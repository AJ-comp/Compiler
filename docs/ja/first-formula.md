ずっと使っている例題文法です。

```
Expr   : Expr '+' Term | Term ;
Term   : Term '*' Factor | Factor ;
Factor : '(' Expr ')' | id ;
```

基本で手作業で求めた答えは `FIRST(Expr) = FIRST(Term) = FIRST(Factor) = { '(', id }` でした。\
ところがその前に — **FIRST が *正確に* 何なのか** から、きちんと定義しておきます。\
(これが抜けると、後の計算規則がただ暗記するだけの呪文になってしまいます。)

---

## 定義 — FIRST とは何か

まず一行で釘を刺しておきます。

> **FIRST(X)** = 記号 `X` が **導出して作り出せる** すべての終端記号列の、**先頭の終端記号** だけを
> ごっそり集めた集合。

肝心な言葉が二つあります — **導出** と **先頭の終端記号**。\
一つずつ見ましょう。

### 「導出(derivation)」 — 規則で展開すること

非終端記号を、その生成規則の右辺に **置き換える** 一歩を *導出* と呼び、矢印 `⇒` で書きます。

`Expr` を一歩ずつ展開してみます。

```
   Expr  ⇒  Term  ⇒  Factor  ⇒  id
        (Expr:Term)  (Term:Factor)  (Factor:id)
```

規則を三回適用して、結局 **終端記号だけからなる文字列** `id` に到達しましたね。\
このように *複数の歩み* で展開することを `⇒*` と書きます → `Expr ⇒* id`(「Expr は id を導出する」)。

### だから FIRST は

`X` を *あらゆる方法で* 最後まで展開して出てくる終端記号列の — **先頭の終端記号** をすべて集めたものです。

`Expr` で実際に見てみると:

```
   Expr ⇒* id …             →  先頭が id   ⇒   id ∈ FIRST(Expr)
   Expr ⇒* ( Expr ) …       →  先頭が (    ⇒   ( ∈ FIRST(Expr)
```

`Expr` が作れる文字列は無数にありますが(`id`, `id + id`, `( id ) * id`, …)、**先頭に来ることが
できる終端記号** は結局 `id` でなければ `(`、この二つだけです。だから:

```
   FIRST(Expr) = { '(', id }
```

記号でもっと厳密に書くとこうなります(`T` = 終端記号の集合):

```
   FIRST(X) = { a ∈ T | X ⇒* a … (a で始まる文字列を導出) }
```

> 📎 **ε(空文字列)が一つだけ追加です。** もし `X` が *何でもないもの* まで導出できるなら
> (`X ⇒* ε`)、**ε も FIRST(X) に入れます。**「X はまるごと消えてしまうこともある」という印です。(私たちの
> 例題にはそういう非終端記号がないので ε は出てきません — でも定義には必ず含まれます。)

まとめると — 終端記号であれ、非終端記号であれ、*記号が複数並んだ列* であれ、**「導出して出てくる文字列の先頭
の終端記号(+必要なら ε)の集合」** が FIRST です。\
これが定義のすべてです。

## 定義どおりに — 導出して自分で求めてみる

では、その定義を *直接適用* して FIRST を求めてみます。\
「導出して先頭の終端記号を集める」を手作業で。\
やさしい `Factor` から。

### Factor — 詰まることなく最後まで展開されます

`Factor` の二つの **生成規則**(`|` で分かれた一行ずつ — `A → α`、詳しくは [Single](deep-single.md))を最後まで導出してみましょう。

```
   Factor ⇒ id              →  先頭の終端記号 :  id
   Factor ⇒ ( Expr )        →  先頭の終端記号 :  (
```

`Factor` が出す文字列の先頭は `id` でなければ `(` — この二つだけです。\
定義どおりに集めると:

```
   FIRST(Factor) = { id, '(' }
```

簡単でしょう?\
`Factor` の中には自分自身が出てこないので、導出がきれいに終わります。

### Term — 展開していくと自分自身がまた出てきます

`Term : Term '*' Factor | Factor`。\
二つの生成規則を導出してみます。

```
   ① Term ⇒ Factor ⇒ … ⇒ id または (          →  先頭 :  id, (
   ② Term ⇒ Term '*' Factor                  →  先頭がまた Term ?!
```

②番が妙です。\
先頭が *自分自身* `Term` なので、先頭の終端記号を知るにはその `Term` を **また** 展開しなければなりません。

```
   Term ⇒ Term '*' Factor
        ⇒ (Term '*' Factor) '*' Factor
        ⇒ …                              ←  展開しても展開しても先頭はずっと Term、終わりが見えない
```

でも結局、先頭の `Term` もいつかは `Factor` 生成規則へと降りていき、そうすると先頭の終端記号はまた `id` や `(` になります。
だから:

```
   FIRST(Term) = { id, '(' }
```

`Expr`(`Expr : Expr '+' Term | Term`)も同じ形なので、`FIRST(Expr) = { id, '(' }` になります。

### ここで引っかかります — 「最後まで」導出できません

`Term` で見たように、**自分自身をくわえていると(再帰)導出が無限に長くなりうるのです。**\
定義は明確なのに(「導出して出てくる先頭の終端記号」)、その導出を *一つひとつ最後まで* 展開するのは、手作業でもコンピュータでも厄介です。
