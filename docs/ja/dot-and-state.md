# ドットと状態 — パーサが「今どのあたり」かを覚えておく方法

前の [FIRST / FOLLOW](first-follow.md) では — パーサが *「始まり/終わり」* を判断するために作っておく **チートシート** を
見ましたね。\
今度は、パーサが入力を読み進めながら **「自分は今どのあたりまで来たんだろう?」** をどうやって覚えておくのかを見ていきましょう。\
(ここでも公式・コードなしで、*感覚* だけで進みます。😀)

## 読んでいくと — 「どこまで読んだか」の印が必要になります

パーサはトークンを **左から一つずつ** 読みます。\
そうしているうちに、ある規則を *始めはしたけれどまだ終わっていない* 状況がしょっちゅう出てきます。

例えば規則 `Expr : Expr '+' Term` をたどっている途中だとしてみましょう。\
`Expr` までは読んでいて、今は `'+'` が来るのを待っているところです。\
この *「どこまで来たか」* をどうやって書き留めておきましょうか?

とても単純な方法があります — **規則の真ん中にドット(`•`)を一つ打つ** のです。

```
   Expr → Expr • '+' Term
```

ドットの **前** は *すでに読んだもの*、ドットの **後ろ** は *まだ読むもの* です。\
上のドットは *「Expr まで読んでいて、次は '+'」* を意味しますね。

> 📖 こうやって **ドットを打った規則** を **LR項目(item)** と呼びます。\
> 名前は大げさですが、たった今見た *「規則 + どこまで」* がすべてです。これからは単に **「項目」** と呼ぶことにします。

ドットはトークンを一つ読むたびに **一マスずつ右へ** 進みます。

```
   Expr → • Expr '+' Term      (まだ何も読んでいない)
   Expr → Expr • '+' Term      (Expr まで読んだ)
   Expr → Expr '+' • Term      ('+' まで読んだ)
   Expr → Expr '+' Term •      (すべて読んだ — さあ一つのかたまりにまとめる番!)
```

ドットが **一番後ろ** に届いたら、その規則を *読み切った* ということです。\
このときがまさに **まとめる(reduce)番** なのです。([FIRST / FOLLOW](first-follow.md) で *「終わったらまとめる」* と言っていた、その
瞬間です。)

## ところで — 可能性はたいてい *複数* あります

ここからもう一歩だけ進みます。\
パーサがある地点に立っているとき、*進行中の規則* が必ず一つだけとは限りません。

`a + a * a` を読んでいて、最初の `a` を `Term` まで見たとしてみましょう。(`a` は名前なので `Factor`、その `Factor` は
`Term` ですね。)\
まさにその瞬間、**二つのことが同時に** 可能になります。

- *「この `Term` 一つで一つのかたまりが終わり」* かもしれないし → `Expr → Term •`
- *「後ろに `* なにか` がもっと付く」* かもしれません → `Term → Term • '*' Factor`

```
   Expr → Term •                 ← ここで終わりかも
   Term → Term • '*' Factor      ← '*' がさらに付くかも
```

こうやって *「今可能な項目たち」* を一つのまとまりに集めたもの — それがパーサの **状態(state)** です。

> 🧭 たとえるなら、状態は **分かれ道に立ったときのナビ画面** のようなものです。\
> *「ここから行ける道はこれら」* を一目で見せてくれるわけですね。\
> そして **次のトークン** がそのうちの一本の道を選んでくれます — `*` ならさらに進み、`+` や入力の終わりならまとめます。\
> (*「まとめてよい次のトークン」* を [FOLLOW](first-follow.md) が教えてくれると言っていたのが、まさにここで使われます。)

## 読むたびに — 状態から状態へ

パーサはトークンを一つ読むたびに **ある状態から別の状態へ移っていきます。**\
まるで *駅から駅へ* 進むようにです。

```
   (開始状態) ──読む──▶ (次の状態) ──読む──▶ (次の状態) ──▶ …
```

ですから LR パーサは実は **「状態と状態のあいだを移動して回る機械」** なのです。\
*どの状態にいるか + 次のトークンが何か* — その二つがあれば次の行動(さらに読む / まとめる)が決まりますね。

> 📦 *この状態が正確に何個あって、どのように作られるのか* には決まった **組み立て手順** があります(ドットを動かし、
> 抜けている可能性を埋めて…)。少し機械的な作業なので、気になるなら **応用コース** の [LR項目](lr-item.md) から
> 一段階ずつ見ていけば大丈夫です。\
> **基本トラックはここまでで十分です** — *「状態 = 今可能なものたちのまとまり、読めば隣の状態へ移る」* という
> 感覚さえつかめれば。🙂
