# Janglim マニュアルへようこそ

Janglim は **.NET 用の LR/LALR パーサ (構文解析器) ジェネレータエンジン**です。\
言葉が難しいですよね?\
**大丈夫です。**\
このマニュアルは
あの難しい単語を一つも知らないという前提で、本当に最初からゆっくり解きほぐしていきます。

## このマニュアルは誰のためのものですか?

**パーサ (構文解析器) が何か、コンパイラがどう動くのか一つも知らなくても大丈夫です。**\
このマニュアルは「なぜこれが
必要なのか」から一歩ずつ、できるだけやさしい言葉で説明します。\
だから読み終える頃には、二つのものが手に残ります。

1. **構文解析 (パーシング) という概念**そのもの — どんなパーサ (構文解析器) を使うにしても一生役立つ知識
2. **Janglim という道具** — そして(望めば)その中身がコードでどのように作られているのかまで

## このマニュアルは二筋に分かれています

負担を減らすために **基本 / 応用** の二つの過程に分けました。

**🟢 基本過程 — 概念だけを気楽に。**\
公式もコードもなしに、*概念と直感* だけをつかみます。\
各ページは:

| 段階 | 何を |
|---|---|
| **① なぜ必要なのか** | これがないと何ができないのか (問題から) |
| **② 何をするのか** | 概念そのもの + 小さな例 |
| **③ プレイグラウンドで見る** | ブラウザで直接動かしながら確認 |

**🔵 応用過程 — 公式 + コード。**\
同じ概念を *どんな文法にも通用する公式(アルゴリズム)* として整理し、
それが **Janglim のコードでどう実装されているのか**まで見ます。\
基本過程の各概念の最後に「→ 応用へ」
というリンクを付けておきます。

> **初めてなら基本過程だけを通して読んでください。**\
> 応用は後で気になったときに見ても十分です。 🙂

## 一つの例で最後まで進みます

マニュアル全体で **たった一つの小さな文法**をずっと使います — 四則演算に似たものです。

```
Expr   : Expr '+' Term | Term ;
Term   : Term '*' Factor | Factor ;
Factor : '(' Expr ')' | id ;
id     := "[a-zA-Z]+" ;
```

そしてこの入力をずっと構文解析していきます: **`a + a * a`**

各チャプターがこの *同じ* 例に理解を一枚ずつ重ねていきます。\
(この文法の読み方は初めて出てくるとき
一緒にゆっくり解きほぐしていくので、前もって心配しなくて大丈夫です。)
