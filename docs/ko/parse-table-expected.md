# 파싱 테이블 · 올 수 있는 토큰

> 🎓 **심화 과정** 이에요.\
> [파싱 테이블 · 만드는 법](parse-table-build.md)에서 표를 다 채우고, 여기까지 LALR 로 정밀하게 다듬어
> 왔죠. 그 표엔 *빈 칸* 도 있었어요 — 만드는 법 장에서 *"빈 칸은 그 입력이 여기서 오류"* 라고 짚었고요.
> 이 장은 그 빈 칸의 *반대편* 이에요 — 같은 줄에서 **채워져 있는 단말 칸들**, 곧 *"여기 올 수 있는
> 토큰"* 을 엔진이 어떻게 뽑아내는지. 코드는 놀랄 만큼 짧아요.

> 📍 **사는 곳** · `LRParsingRowDataFormat.PossibleTerminalSet` · `…/Parsers/Datas/LR/…`

## 빈 칸이 있는 줄 하나

[완성된 표](parse-table-build.md)에서 `I₆` 줄을 떼어 볼게요. (상태로 치면 `Expr` 하나에 `'+'` 를 막
shift 한 자리, 즉 `Expr +` 예요.)

| 상태 | `id` | `'+'` | `'*'` | `'('` | `')'` | `$` |
|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| `I₆` | s5 |   |   | s4 |   |   |

이 줄에서 행동이 들어 있는 **단말** 칸은 딱 둘, `id`(s5)와 `'('`(s4)예요. 나머지 `'+'`·`'*'`·`')'`·`$`
는 다 비어 있죠.

그러니 이 상태에서 다음 토큰이 가령 `'+'` 라면 — 그 칸이 비었으니 **오류** 예요. 그럼 대신 뭐가 올 수
있었을까요? 채워진 칸을 보면 `id` 와 `(` — 그게 `I₆` 에서 **올 수 있는 토큰 `{ id, '(' }`** 예요.
코드로 옮기면 딱 한 문장, *"이 줄에서 행동이 있는 단말 칸을 모아라."*

## `PossibleTerminalSet` — 줄 하나를 훑는다

```csharp
public override TerminalSet PossibleTerminalSet
{
    get
    {
        TerminalSet result = new TerminalSet();

        foreach (var valueSet in this.MatchedValueSet)                      // ← 이 상태(줄)의 칸을 하나씩
        {
            if (valueSet.Value[0].Direction == ActionDir.Failed) continue;  // ← 행동 없는(막힌) 칸은 건너뜀
            if (valueSet.Key is Terminal) result.Add(valueSet.Key as Terminal);   // ← 단말 칸만 담음
        }

        return result;
    }
}
```

`MatchedValueSet` 이 바로 *그 상태의 한 줄* 이에요 — 기호(열)마다 어떤 행동(`s5`·`r2`…)이 붙어 있는지를
담고 있죠. 그 줄을 훑으면서 **행동이 없는 칸(`Failed`)은 건너뛰고, 단말인 칸만** 담으면, 남는 게 곧
*"이 자리에서 올 수 있는 토큰 전부"* 예요. `I₆` 줄에 이걸 돌리면 정확히 `{ id, '(' }` 가 나오고요.

> 💡 `GOTO` 쪽(비단말 칸)은 `is Terminal` 에서 걸러져요 — 사용자에게 *"이게 와야 한다"* 고 보여줄 건
> *입력하는 토큰(단말)* 이지 파서가 안에서 묶어내는 비단말이 아니니까요. (비단말용은
> `PossibleNonTerminalSet` 로 따로 있어요.)

## 말로 한 정의가 코드에 그대로

*"막힌 상태의, 채워진 단말 칸들"* — 이 한 줄짜리 정의가 코드에선 **`foreach` 한 바퀴** 예요. 그래서
파서는 오류가 난 순간 이 집합을 이미 손에 쥐고 있어요. 실제로 막히면 이 값이 그 자리의 파싱 단위로
복사되고(`LRParser` 의 방향을 정하는 지점), 엔진 안에는 이걸 끼워 넣을 메시지 틀 — *"(여기엔 `{…}` 가
와야 해요)"* — 까지 마련돼 있어요.

## 지금은, 그리고 앞으로

다만 **기본 에러 핸들러** 는 아직 이 집합을 메시지로 *꺼내 쓰진 않아요.* 지금은 위치가 담긴 오류 하나 —
*"`+` 는 여기 올 수 없는 토큰"* — 만 남기고 멈추죠. 하지만 `PossibleTerminalSet` 은 이미 계산돼 *표 안에
들어 있으니*, 그걸 *"`id` 나 `(` 가 와야 합니다"* 같은 친절한 메시지로 바꾸는 건 핸들러 한 겹만 씌우면
되는 일이에요.

> 🧭 이 집합은 *오류 복구* 의 열쇠이기도 해요 — 막힌 자리에서 *"올 수 있는 토큰이 다시 나올 때까지"*
> 입력을 건너뛰거나 스택을 되감는 식(panic mode)으로요. 그건 다음 이야기로 남겨둘게요.
